03
系统目标、意义与范围

🎯 系统目的

  • 帮助非金融专业人士轻松读懂并分析上市公司财务报表
  • 充当"审计助理"，自动抓取高危指标并拉响警报
  • 所有数据提供PDF页码溯源，支持用户一键验证

📏 系统范围

  • 聚焦中文上市公司年报/季报 PDF 格式
  • 五大功能：PDF解析 → 向量检索 → 智能问答 → 指标计算 → 摘要生成

💡 系统意义

  • 降低投资门槛
    让普通投资者也能像专业分析师一样审阅财报
  • 消除算术幻觉
    Agent沙盒隔离计算，彻底杜绝大模型数值伪造
  • 可验证可信任
    RAG全链路溯源，每个结论指向原始PDF页码
目标用户
缺乏专业审计能力的普通股票投资者

04
功能需求 — 核心用例描述
用例名称 前置条件 主要流程
PDF智能解析 用户已上传PDF文件 系统采用混合解析策略：普通页面使用PyMuPDF本地提取，表格页面送LlamaParse高精度解析，自动识别有边框和无边框表格，结果本地缓存
RAG溯源问答 PDF解析完成，向量索引已构建 用户提问 → 系统在向量库中检索最相关文本片段 → 组装上下文发送给LLM → 生成答案并附带源文档页码引用 → 用户可一键验证
Agent沙盒计算 用户提出涉及财务指标计算的问题 Agent通过Plan-Execute-Observe动态规划 → 从上下文提取计算参数 → 注入Python确定性沙盒执行计算 → 返回精确数值结果
一键排雷 PDF上传完成 系统自动后台巡检核心风险科目 → 在界面顶部生成可视化风险指示灯（高危/正常）→ 变"被动提问"为"主动防御"

05
性能需求与接口需求

⚡ 性能需求

  • PDF解析性能
    100页财报本地解析 <30秒，含表格页面启用LlamaParse高精度模式
  • 问答响应时间
    RAG检索 + LLM生成首字响应 <5秒，完整回答流式输出
  • 向量检索效率
    FAISS索引在100页财报的向量库中，Top-K检索响应 <1秒
  • 计算准确性
    Agent沙盒计算结果精确到小数点后2位，财务指标计算零幻觉

🔌 接口需求

  • 用户接口
    Streamlit Web界面，文件上传、聊天对话、可视化排雷大盘
  • 外部接口
    DeepSeek LLM API（文本生成）
    LlamaParse API（高精度表格解析）
  • 软件接口
    FastAPI RESTful API
    前端与后端通过HTTP通信
06
其它需求
🔒
安全性
用户上传的财报文件在本地完成解析和向量化，仅检索查询通过API调用云端LLM。Agent计算在隔离沙盒中执行，防止恶意代码注入。
🔧
可维护性
采用模块化架构：PDF解析、向量检索、Agent计算、前端UI 各模块独立。核心LLM支持DeepSeek及OpenAI兼容接口替换。
📦
可移植性
支持Docker一键部署，也可本地Poetry开发环境运行。依赖Python 3.11，推荐配备NVIDIA GPU以提升推理性能。
✨
易用性
Web交互界面简洁直观，用户只需上传PDF即可使用。系统自动完成解析、索引构建、排雷检测，无需手动配置。
