# Docker 环境测试教学指南

> **面向测试人员（队友）** —— 在 Docker 容器中运行自动化测试 + 手工测试前端功能。

---

## 一、环境准备

### 前置要求

1. **安装 Docker Desktop**
   - 下载地址：https://www.docker.com/products/docker-desktop
   - 安装后重启电脑，确保 Docker Desktop 正在运行（系统托盘有鲸鱼图标）

2. **获取项目代码**
   ```bash
   git clone <项目仓库地址>
   cd financial-analysis-AI-assistant
   ```

3. **确认 `.env` 文件存在**
   - 项目根目录应有一个 `.env` 文件（已由负责人配置好）
   - 如果没有，找负责人要一份

---

## 二、启动服务（3 步）

### 第 1 步：构建 Docker 镜像

```bash
docker compose build --no-cache
```

> ⏳ 这一步比较久（约 10-20 分钟），因为要下载 Python 包和模型。
> **只需要构建一次**，以后不用再跑。

### 第 2 步：启动服务

```bash
docker compose up -d
```

> `-d` 表示后台运行。启动后等 10 秒左右。

### 第 3 步：验证服务已启动

- 浏览器打开 **http://localhost:8501** ，应该能看到前端界面
- 浏览器打开 **http://localhost:8000** ，应该返回 `{"status": "ok"}`

---

## 三、运行自动化测试

**另开一个新的命令行窗口**，进入项目目录，执行：

```bash
docker compose exec backend poetry run pytest tests/ -v
```


**看到 `168 passed` 就说明全部通过 ✅**

### 如果出现 FAILED

把**完整的红色报错**截图发给项目负责人。

### SKIPPED 项可以忽略

有些测试因为 Docker 里缺少 PyMuPDF 等库会被跳过（显示 `SKIPPED`），**这不算失败，忽略就行**。


## 六、停止和重启服务

```bash
# 停止服务（容器还在）
docker compose stop

# 停止并删除容器
docker compose down

# 重新启动
docker compose up -d

# 查看后端日志（排查问题时用）
docker compose logs backend

# 查看前端日志
docker compose logs frontend
```

## 七、常见问题

| 问题 | 原因 | 解决办法 |
|------|------|----------|
| `docker compose` 命令不存在 | Docker Desktop 没装 | 安装 Docker Desktop |
| `docker compose build` 超时 | 网络问题 | Dockerfile 已配国内镜像源，如仍超时换网络重试 |
| `docker compose exec` 报错 "no such container" | 容器没启动 | 先跑 `docker compose up -d` |
| 测试出现 `FAILED` | 代码或环境问题 | 把完整报错截图发给我 |
| 测试出现 `SKIPPED` | 某些库不可用 | **不算失败**，忽略即可 |
| 浏览器 8501 打不开 | 前端容器没启动 | 跑 `docker compose up -d`，等10秒再试 |
| 前端报"连接错误" | 后端容器没启动 | 跑 `docker compose logs backend` 查日志 |
| `poetry run pytest` 报错找不到 pytest | 容器内依赖没装好 | 重新 `docker compose build --no-cache` |
