## 三、RAG 缓存优化（已完成）

### 3.1 问题
每次上传同一 PDF 都会重建向量库，耗时约 30 秒。

### 3.2 解决方案
在 `build_vector_store()` 中增加本地索引检测逻辑：
- 检查 `faiss_index/` 目录是否存在
- 如果存在则加载已有索引，跳过重建
- 新增 `_rebuild_page_num_map()` 函数在加载索引时重建页码映射

### 3.3 修改文件
- `src/.../services/rag_service.py` - `build_vector_store()` 函数

### 3.4 效果
| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 首次上传 | 32 秒 | 32 秒 |
| 二次上传 | 32 秒 | **6.5 秒** |
| 提升 | - | **80%** |

## 五、数据流

```
1. 上传 PDF
   └─→ /upload 保存 PDF 到 disk + 解析文本 + 构建按页切分的 RAG

2. 用户提问
   └─→ /chat
       ├─→ query_rag_with_source() 返回 {"context": "...", "page_num": 5}
       ├─→ run_agent_query() 生成回答
       └─→ 返回 {"answer": "...", "source_page": 5}

3. 前端展示
   └─→ st.image(f"/highlight?page=5&x=0&y=0&w=600&h=800")
       └─→ 显示带红框的财报截图
```
