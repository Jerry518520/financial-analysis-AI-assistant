## 二、Bug 修复（已完成）

### 2.1 torch DLL 崩溃

- **问题**：`import torch` 时进程崩溃，错误码 `0xc0000139`
- **原因**：torch 2.10.0 与 torchvision 0.21.0 版本不兼容
- **修复**：安装 torchvision 0.22.0，自动将 torch 降级到兼容的 2.7.0+cpu

| 指标 | 修复前 | 修复后 |
|------|--------|--------|
| torch 版本 | 2.10.0（崩溃） | 2.7.0+cpu（正常） |
| torchvision 版本 | 0.21.0 | 0.22.0+cpu |

### 2.2 RAG 测试全局状态污染

- **问题**：RAG 服务 29 个测试大面积失败
- **原因**：模块级全局变量 `PAGE_NUM_MAP` 在测试之间互相污染
- **修复**：在 `tests/services/test_rag_service.py` 添加 autouse fixture，每个测试前后重置 `PAGE_NUM_MAP = {}`

### 2.3 Highlight 测试库依赖缺失

- **问题**：`/highlight` 相关测试 import 报错
- **原因**：高亮功能懒加载 `fitz`（PyMuPDF）和 `PIL`，未安装时 import 链断裂
- **尝试**：`sys.modules` mock + `importlib.reload`，但破坏 FastAPI TestClient 绑定的 app 实例
- **修复**：库不可用时 `pytest.skip`，CI 环境中自动跳过

### 2.4 上传测试污染后续测试

- **问题**：上传测试后"无 PDF 时返回 404"测试假阳性通过
- **原因**：上传测试设置全局变量 `CURRENT_PDF_PATH`，后续测试读到该值
- **修复**：在 `tests/api/test_api_endpoints.py` 添加 fixture，测试后重置 `CURRENT_PDF_PATH = None`

## 三、测试框架设计说明

### 3.1 设计原则

| # | 原则 | 实现方式 |
|---|------|----------|
| 1 | 隔离性 | 每个 test 不依赖其他 test，fixture 重置全局状态 |
| 2 | 无需真实服务 | 全部使用 FastAPI TestClient，不启动后端 |
| 3 | 无需真实 API Key | conftest.py Mock 所有 LLM API Key |
| 4 | 边界值覆盖 | 每个计算函数覆盖零值、负值、极大值 |
| 5 | 优雅降级 | 缺少可选依赖时 skip 而非 fail |

### 3.2 运行命令

| 命令 | 说明 |
|------|------|
| `poetry run pytest tests/ -v` | 运行全部测试（详细输出） |
| `poetry run poe test` | 通过 poe 运行 |
| `poetry run poe test-cov` | 运行并生成覆盖率报告 |
