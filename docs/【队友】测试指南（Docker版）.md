# AI财报分析助手 - 测试指南（Docker版）

> **测试人员操作手册** —— 使用Docker环境完成自动化测试和手工测试

---

## 一、准备工作

### 1. 安装Docker Desktop
- 下载地址：https://www.docker.com/products/docker-desktop
- 安装后重启电脑，确保系统托盘有鲸鱼图标

### 2. 获取项目代码
```bash
git clone https://github.com/Jerry518520/financial-analysis-AI-assistant.git
cd financial-analysis-AI-assistant
```

### 3. 确认环境文件
- 项目根目录应有 `.env` 文件（找负责人要）

---

## 二、启动服务（3步）

### 第1步：构建镜像（只需一次，约10-20分钟）
```bash
docker compose build --no-cache
```

### 第2步：启动容器
```bash
docker compose up -d
```

### 第3步：验证服务
- 浏览器打开 http://localhost:8501 ，应看到前端界面
- 浏览器打开 http://localhost:8000 ，应返回 `{"status": "ok"}`

### 6.4 评分标准

#### 评分维度（满分100分）

| 维度 | 权重 | 说明 |
|------|------|------|
| 数值准确度 | 30% | 数字是否正确提取/计算 |
| 计算准确度 | 20% | 计算过程和结果是否正确 |
| 分析完整性 | 20% | 是否覆盖所有问点 |
| 逻辑清晰度 | 15% | 回答是否条理清晰 |
| 回答速度 | 15% | 响应时间（主观评价） |

#### 星级标准

| 星级 | 分数 | 标准 |
|------|------|------|
| ⭐⭐⭐⭐⭐ | 90-100 | 完美回答，数据准确 |
| ⭐⭐⭐⭐ | 75-89 | 正确，少量缺失 |
| ⭐⭐⭐ | 60-74 | 基本正确，有错误 |
| ⭐⭐ | 40-59 | 部分正确，较多错误 |
| ⭐ | 0-39 | 错误或无法回答 |

### 6.5 对比方法

对每个模型，逐题执行以下步骤：

1. **上传同一份财报 PDF**（如果模型支持文件上传）
2. **粘贴统一提示词 + 测试问题**
3. **记录完整回答**（原文复制到记录表）
4. **计时**（从发送问题到收到完整回答）
5. **按评分标准打分**

### 6.6 对比分析要点

测试完成后，需要分析以下方面：

1. **数值准确性对比**：哪个模型提取的数字最准确？本项目有 RAG 检索 + 结构化计算，应显著优于纯大模型的"幻觉"
2. **计算能力对比**：哪个模型的计算结果最正确？本项目有专门的财务计算工具函数，计算不会出错
3. **溯源能力对比**：本项目可以返回"来源页码 + 溯源图片"，其他模型能否提供？
4. **回答速度对比**：本项目 vs 其他模型的响应时间
5. **综合评分排名**：按总分排出名次

> **重点关注**：本项目是 RAG（检索增强生成）+ Agent（工具调用）架构，
> 相比纯大模型的"裸答"，在**数值准确性**和**可溯源**两个维度上应该有明显优势。
> 对比时要突出这两个差异。
