## 2 任务概述

### 2.1 目标

本软件主要作用是帮助非金融专业人士轻松读懂并分析上市公司PDF格式的财务报表，具有以下特点：

①本系统为Web应用系统，提供直观的交互界面，操作简洁方便。

②为了实现财报的智能分析，系统采用了RAG检索增强技术和AI Agent技术，能够准确回答财务问题并自动进行指标计算。

③本系统的问答功能可以回答具体的财务数据问题，如营业收入、净利润、资产负债率等。

④本系统的计算功能提供了自动调用工具进行精确计算的方式，包括同比增长率、毛利率、ROE等指标的计算。

⑤本系统包含"PDF解析"、"向量检索"、"智能问答"、"指标计算"、"摘要生成"等功能。

### 2.2 用户的特点

本软件使用用户为非金融背景的专业人士。

| 用户类型 | 特点 |
|----------|------|
| 普通用户 | 只能通过自然语言提问获取财报数据和分析结果，无需财务专业知识 |
| 投资者 | 需要快速分析目标公司的财务健康状况，对数据准确性要求较高 |
| 企业管理人员 | 需要了解竞争对手或行业的财务表现，期望获得简明扼要的财务摘要 |
| 学生研究者 | 学习财务分析知识，需要从财报中提取数据进行研究分析 |

### 2.3 假定和约束

2.3.1开发环境需要Python 3.11、Docker等技术支持

2.3.2系统依赖外部API服务（DeepSeek LLM、LlamaParse文档解析），需要网络连接

2.3.3本地硬件推荐配备NVIDIA GPU以提升向量计算和模型推理性能

2.3.4系统主要针对中文财报进行优化，英文财报支持度待验证

## 3 需求规定

### 3.1 对功能的规定

财报AI分析助手主要实现PDF解析功能、向量检索功能、智能问答功能、指标计算功能、摘要生成功能

3.1.1PDF解析功能

&nbsp;&nbsp;用户通过Web界面上传PDF财报文件，系统采用混合解析策略，普通页面使用PyMuPDF本地提取，表格页面送LlamaParse高精度解析，自动识别有边框和无边框表格，解析结果本地缓存避免重复解析。

3.1.2向量检索功能

&nbsp;&nbsp;解析完成后，系统自动将文本内容向量化并构建FAISS向量索引，使用MarkdownTextSplitter进行智能切分，使用HuggingFace Embeddings模型将文本块转换为向量，在向量库中检索与问题最相关的文本片段。

3.1.3智能问答功能

&nbsp;&nbsp;将检索到的上下文与用户问题组合，发送给LLM生成最终答案，支持数值查询、计算题回答、分析类问题回答等多种问题类型。

3.1.4指标计算功能

&nbsp;&nbsp;Agent自动调用计算工具完成财务指标计算，包括同比增长率计算（公式：(本期值-上期值)/|上期值|）、利润率计算（公式：利润/营收）、净资产收益率ROE计算（公式：净利润/净资产）。

3.1.5摘要生成功能

&nbsp;&nbsp;用户点击生成核心摘要按钮后，系统执行多查询检索，综合多个财务主题的检索结果，调用LLM生成结构化摘要，包含核心财务指标、经营亮点、风险提示、未来展望。

### 3.2 对性能的规定

3.2.1精度

&nbsp;&nbsp;由于采用RAG检索技术和LLM生成技术，数据准确性受文档质量和检索效果影响，对于明确存在于文档中的结构化数据，准确率应达到90%以上。财务指标计算采用浮点数运算，结果保留4位小数，百分比保留2位小数。

3.2.2时间特性要求

| 操作 | 预期时间 |
|------|----------|
| PDF解析（20页以内） | 30秒-2分钟 |
| 向量索引构建 | 10-30秒 |
| RAG检索响应 | 小于1秒 |
| LLM回答生成 | 3-15秒 |
| 摘要生成 | 15-30秒 |

3.2.3灵活性

&nbsp;&nbsp;本系统支持模型可替换，核心LLM支持DeepSeek及OpenAI兼容接口。解析策略可调，LlamaParse页面处理数量可通过参数配置。检索参数可调，top_k检索数量、chunk_size等参数可根据实际效果调整。部署方式灵活，支持Docker一键部署和本地Poetry开发两种模式。
